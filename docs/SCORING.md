# 评分规则文档

## 概述

本项目采用多维度评分模型，总分 100 分，由 4 个一级维度组成。评分规则基于 `discuss004.md` 定义，实现于 `backend/app/config/scoring.py`。

## 评分框架

```
总分 = ICP 匹配度 + 需求信号 + 可触达可行性 - 风险扣分
     = (0-35分)    + (0-35分)  + (0-15分)     - (0-15分)
     = 0-100 分
```

---

## 一、ICP 匹配度评分（0-35分）

ICP (Ideal Customer Profile) 匹配度评估企业是否符合目标客户画像。

| 子维度 | 满分 | 说明 |
|--------|------|------|
| 行业匹配 | 12分 | 是否属于目标行业或相邻行业 |
| 企业规模匹配 | 8分 | 员工规模、组织复杂度是否适配 |
| 业务场景匹配 | 10分 | 是否存在我方产品对应场景 |
| 企业类型匹配 | 5分 | 是否适合当前销售模式 |

### 行业匹配评分规则

| 条件 | 分数 |
|------|------|
| 明确属于核心目标行业 | 12分 |
| 属于相邻可拓展行业 | 8分 |
| 行业关联弱但可能可做 | 4分 |
| 明显不匹配 | 0分 |

### 企业规模匹配评分规则

| 条件 | 分数 |
|------|------|
| 完全符合目标规模带 | 8分 |
| 略大或略小，但仍可服务 | 5分 |
| 偏差明显，但理论可做 | 2分 |
| 明显不适配 | 0分 |

### 业务场景匹配评分规则

| 条件 | 分数 |
|------|------|
| 明确存在核心业务场景 | 10分 |
| 存在部分可切入场景 | 6分 |
| 场景模糊，仅弱相关 | 3分 |
| 无明显适配场景 | 0分 |

### 企业类型匹配评分规则

| 条件 | 分数 |
|------|------|
| 与我方销售模式高度适配 | 5分 |
| 存在一定适配性 | 3分 |
| 流程复杂或成交门槛高 | 1分 |
| 基本不适配 | 0分 |

### 客户匹配等级映射

| 分数范围 | 等级 |
|---------|------|
| 26-35分 | high（高匹配） |
| 15-25分 | medium（中匹配） |
| 0-14分 | low（低匹配） |

## 二、需求信号评分（0-35分）

评估企业是否存在明确的需求信号。

| 子维度 | 满分 | 说明 |
|--------|------|------|
| 动态新鲜度 | 10分 | 最近 3-6 个月是否有变化 |
| 信号强度 | 15分 | 是否直接指向采购/建设需求 |
| 多信号一致性 | 10分 | 是否多个来源互相印证 |

### 动态新鲜度评分规则

| 条件 | 分数 |
|------|------|
| 近 3 个月有强相关动态 | 10分 |
| 近 6 个月有相关动态 | 7分 |
| 近 12 个月有较弱动态 | 4分 |
| 无有效近期动态 | 0分 |

### 信号强度评分规则

| 条件 | 分数 |
|------|------|
| 有直接需求信号 | 15分 |
| 有较强间接信号 | 10分 |
| 有弱信号 | 5分 |
| 几乎无信号 | 0分 |

### 多信号一致性评分规则

| 条件 | 分数 |
|------|------|
| 3个及以上不同来源互相印证 | 10分 |
| 2个来源相互支持 | 6分 |
| 仅单一来源 | 3分 |
| 没有可靠证据链 | 0分 |

### 商机等级映射

| 分数范围 | 等级 |
|---------|------|
| 26-35分 | high（高商机） |
| 15-25分 | medium（中商机） |
| 0-14分 | low（低商机） |

---

## 三、可触达可行性评分（0-15分）

评估是否能够有效触达目标客户。

| 子维度 | 满分 | 说明 |
|--------|------|------|
| 目标角色清晰度 | 5分 | 是否知道找谁 |
| 切入场景清晰度 | 5分 | 是否知道聊什么 |
| 组织路径可推断性 | 5分 | 是否能大致判断决策链 |

### 各子维度评分规则

| 条件 | 分数 |
|------|------|
| clear（清晰） | 5分 |
| inferable/general/partial（可推断/一般/部分） | 3分 |
| vague/weak（模糊/弱） | 1分 |
| unknown/none（未知/无） | 0分 |

---

## 四、风险扣分规则（0-15分）

根据风险类型和等级进行扣分。

### 风险类型分类

**高影响风险**：
- legal（法律风险）
- compliance（合规风险）
- financial（财务风险）

**中影响风险**：
- procurement_complexity（采购复杂）
- organization_instability（组织不稳定）
- long_decision_cycle（决策周期长）

**中低影响风险**：
- unclear_demand（需求不明确）
- low_budget_probability（预算概率低）
- reputation（声誉风险）
- other（其他）

### 扣分规则

| 风险等级 | 高影响风险 | 中影响风险 | 中低影响风险 |
|---------|-----------|-----------|-------------|
| high | 5分 | 4分 | 3分 |
| medium | 4分 | 3分 | 2分 |
| low | 2分 | 1分 | 1分 |

**注意**：总扣分上限为 15 分。

---

## 五、总分映射规则

| 总分 | 优先级 | 说明 |
|------|--------|------|
| 75-100 | P1 | 高优先级，建议尽快跟进 |
| 55-74 | P2 | 中优先级，建议进入跟进池 |
| 35-54 | P3 | 低优先级，建议保留观察 |
| 0-34 | discard | 放弃，不建议继续投入 |

---

## 六、兜底规则

### 强制 discard 条件

- customer_fit_level = low 且 opportunity_level = low
- 存在 high 级 financial/legal/compliance 风险且无明确业务机会
- 企业类型明显不适配当前产品交付方式
- 信息极度不足，无法识别行业、规模和场景

### 强制 P1 条件

- customer_fit_level = high 且 opportunity_level = high
- 近3个月内存在明确数字化/采购/招聘相关强信号，且场景与产品高度匹配
- 已有多个来源共同证明企业正在进行组织扩张或系统建设

### 降级条件

即使总分高，也建议降级到 P2 或 P3：
- 采购链明显复杂，且当前无法识别关键角色
- 需求信号较强，但预算/时机不明
- 企业确有需求，但已高度怀疑被现有成熟系统锁定
- 风险扣分较高（>=10）

## 七、辅助函数

```python
# 计算风险扣分
def get_risk_penalty(risk_type: str, risk_level: str) -> int

# 计算总风险扣分（上限15分）
def calculate_total_risk_penalty(risks: List[Dict]) -> int

# 计算总分
def calculate_total_score(
    icp_fit_score: int,
    demand_signal_score: int,
    engagement_feasibility_score: int,
    risk_penalty_score: int
) -> int

# 获取跟进优先级
def get_follow_up_priority(total_score: int) -> str

# 获取客户匹配等级
def get_customer_fit_level(icp_fit_score: int) -> str

# 获取商机等级
def get_opportunity_level(demand_signal_score: int) -> str
```
