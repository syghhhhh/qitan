# 项目架构文档

## 概述

企业背调智能体 MVP 是一个面向企业商务岗位的背调工具，帮助商务人员快速了解目标企业，生成企业画像、需求信号、风险提示、联系人建议和沟通话术。

## 模块说明

### Backend 模块

| 模块路径 | 职责 |
|---------|------|
| `main.py` | FastAPI 应用入口，定义路由和中间件 |
| `schemas/` | Pydantic 数据模型定义 |
| `prompts/` | LLM Prompt 模板 |
| `services/` | 业务逻辑层 |
| `config/` | 配置文件（评分规则等） |

### Frontend 模块

| 文件 | 职责 |
|------|------|
| `index.html` | 页面结构：输入区域 + 结果展示区域 |
| `style.css` | 卡片式布局样式 |
| `app.js` | 表单交互、API 调用、结果渲染 |

## 数据流

```
用户输入 → Frontend (app.js)
                │
                │ POST /analyze
                ▼
          Backend (main.py)
                │
                │ 调用 mock_analyzer
                ▼
          Services (mock_analyzer.py)
                │
                │ 返回 DueDiligenceOutput
                ▼
          Frontend 渲染结果
```

## 未来扩展方向

1. **LLM 集成**：将 Mock 分析替换为真实 LLM 调用
2. **数据采集**：接入企业信息 API（天眼查、企查查等）
3. **用户系统**：添加登录、历史记录功能
4. **缓存层**：Redis 缓存分析结果
5. **任务队列**：异步处理大规模分析任务

## 设计原则

- **MVP 优先**：先实现核心功能，再迭代扩展
- **Mock 驱动**：前后端联调时使用 Mock 数据，后续替换为真实实现
- **Schema 优先**：数据模型定义先行，确保前后端一致
- **配置外置**：评分规则、Prompt 模板等可配置，便于调整
