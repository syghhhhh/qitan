# 一、MVP 目标定义

先明确：  
**MVP 不追求“全自动、全行业、全准确”，只追求“对商务真的有用”。**

所以 MVP 的目标应该是：

> 输入一个目标企业名称 + 我方产品/服务信息，输出一份可供商务快速使用的企业背调摘要，并给出初步沟通切入点、推荐联系人和首轮话术建议。

---

## MVP 解决的核心问题

对商务来说，MVP 至少要回答 5 个问题：

1. **这家公司是谁？**
2. **最近有没有值得关注的动态？**
3. **它可能有没有需求？**
4. **我应该优先找谁？**
5. **我第一次该怎么开口？**

---

## MVP 的边界

建议你先把边界收紧，不然很容易做炸。

### MVP 先做的
- 面向单个企业的背调
- 基于公开信息生成摘要
- 输出结构化背调结果
- 输出基础的商务建议和话术
- 支持人工补充信息后重新生成

### MVP 暂时不做
- 自动大批量扫库
- 非常复杂的客户评分体系
- 很深的竞品情报分析
- 高精度决策链挖掘
- 完整 CRM 自动化闭环
- 行业特化到特别深

---

# 二、MVP 用户与使用场景

---

## 目标用户

### 1. 一线商务 / 销售
诉求：
- 快速了解陌生企业
- 减少查资料时间
- 生成首轮触达话术

### 2. 销售经理 / 商务主管
诉求：
- 看背调结论是否靠谱
- 判断是否值得投入跟进
- 给团队标准化方法

### 3. 售前 / 解决方案顾问
诉求：
- 在会前快速建立企业认知
- 准备会议沟通提纲

---

## MVP 典型场景

### 场景 1：开发新客户前
输入企业名，快速出一页摘要，判断是否值得打。

### 场景 2：首次联系前准备
生成针对目标企业的开场话术、邮件话术、电话切入点。

### 场景 3：领导临时要分析某家公司
5 分钟内生成结构化报告，不需要人工到处搜资料。

# 三、MVP 核心功能模块

我建议 MVP 只做 5 个核心模块。

---

## 模块 1：企业信息输入模块

这是入口模块。

### 输入项建议

#### 必填
- 目标企业名称

#### 选填
- 企业官网
- 所属行业
- 我方产品/服务描述
- 我方目标客户类型
- 本次跟进目标
  - 获客破冰
  - 约首次会议
  - 会前准备
  - 深度方案沟通
- 目标联系人角色
  - CEO
  - 销售负责人
  - 运营负责人
  - IT/数字化负责人
  - HR
  - 采购
- 用户手动补充背景
  - 例如：“我们卖 CRM 系统”
  - “想切销售管理场景”
  - “已经加上对方微信了”

### 为什么这个模块很重要
因为**同一家企业，在不同销售场景下，输出结果应该不同**。

比如：
- 做 CRM 的，会关注销售管理和客户数据
- 做培训的，会关注组织扩张和人才建设
- 做 AI 办公的，会关注流程效率和知识管理

---

## 模块 2：企业信息采集与摘要模块

MVP 不一定要一开始做成复杂爬虫系统，可以先做**可控的数据接入 + LLM 摘要**。

### 建议采集的信息类型
- 企业基础介绍
- 官网简介
- 新闻/动态
- 招聘信息
- 公众号/公开文章
- 工商基础信息
- 风险信息（基础级）

### 这一模块输出什么
输出结构化基础信息：

- 企业基本画像
- 最近动态
- 招聘/扩张/融资等信号
- 风险类公开信号

### 工程建议
MVP 阶段可采用：

#### 方案 A：半自动
- 用户输入企业名
- 系统通过搜索/API 拉取若干公开文本
- LLM 进行清洗和归纳

#### 方案 B：人工可补录
- 系统支持用户贴入官网介绍/新闻链接/文本
- 模型统一抽取成结构化数据

如果你现在数据源不稳定，**强烈建议支持“人工补充文本”**，这样 MVP 更稳。

---

## 模块 3：结构化字段抽取模块

这个模块是 MVP 的关键。  
你的系统不能只输出一段自然语言，而必须沉淀出字段。

### 作用
把原始信息抽取成统一结构，供后续判断和话术生成使用。

### MVP 最低要求
- 字段标准化
- 来源记录
- 时间记录
- 区分事实和推断

---

## 模块 4：商务判断模块

这是从“信息”变成“建议”的核心层。

### MVP 先做 4 个判断能力

#### 1. 企业画像判断
- 企业属于什么类型
- 是否符合目标客户画像
- 当前大致发展阶段

#### 2. 需求信号判断
- 是否存在潜在需求信号
- 需求更可能偏向哪类场景

#### 3. 联系人建议
- 优先接触哪个部门/角色
- 为什么

#### 4. 跟进优先级建议
- 高 / 中 / 低
- 给出简要依据

---

## 模块 5：商务输出生成模块

这个模块直接面向用户。

### 建议输出 4 类内容

#### 1. 背调摘要
给用户 1 分钟看完的结论。

#### 2. 商机分析
告诉用户“为什么值得跟/不值得跟”。

#### 3. 联系人与切入点建议
告诉用户找谁、从哪里切入。

#### 4. 话术模板
输出可直接改一改就能用的话术。

---

# 四、MVP 字段设计

下面是我建议你直接采用的 **MVP 字段 Schema**。  
我会按工程好落地的思路设计，不求完美，但求好用。

---

## 1. 输入层字段

```json
{
  "input": {
    "company_name": "",
    "company_website": "",
    "user_company_product": "",
    "user_target_customer_profile": "",
    "sales_goal": "",
    "target_role": "",
    "extra_context": ""
  }
}
```

### 字段说明
- `company_name`: 目标企业名称
- `company_website`: 企业官网，可选
- `user_company_product`: 我方卖什么
- `user_target_customer_profile`: 我方理想客户画像
- `sales_goal`: 本次目标，例如“首次破冰”
- `target_role`: 希望接触的角色
- `extra_context`: 用户补充说明

---

## 2. 企业基础画像字段

```json
{
  "company_profile": {
    "company_name": "",
    "short_name": "",
    "industry": [],
    "company_type": "",
    "founded_year": null,
    "headquarters": "",
    "business_scope": [],
    "main_products_or_services": [],
    "estimated_size": "",
    "region_coverage": [],
    "official_website": "",
    "official_accounts": []
  }
}
```

### 字段解释
- `industry`: 行业标签，可多值
- `company_type`: 民营/国企/外企/上市公司/集团子公司等
- `business_scope`: 主营业务概述
- `estimated_size`: 小微/中型/大型/未知，或者员工规模区间
- `region_coverage`: 覆盖地区

---

## 3. 近期动态字段

```json
{
  "recent_developments": [
    {
      "date": "",
      "type": "",
      "title": "",
      "summary": "",
      "source": "",
      "confidence": 0.0
    }
  ]
}
```

### `type` 建议枚举
- news
- financing
- hiring
- expansion
- new_product
- bidding
- partnership
- digital_transformation
- management_change
- other

### 为什么这样设计
因为后面做规则判断时，可以直接按事件类型做推断。

---

## 4. 需求信号字段

```json
{
  "demand_signals": [
    {
      "signal_type": "",
      "signal": "",
      "evidence": "",
      "inference": "",
      "strength": "high",
      "source": "",
      "date": ""
    }
  ]
}
```

### `signal_type` 示例
- recruitment_signal
- expansion_signal
- digitalization_signal
- growth_signal
- management_signal
- cost_reduction_signal
- compliance_signal
- customer_operation_signal

### 字段意义
- `signal`: 事实信号
- `evidence`: 证据原文/摘要
- `inference`: 商务推断
- `strength`: high / medium / low

### 这个字段特别重要
因为它把“事实”和“推断”连接起来了。

---

## 5. 组织与关键角色字段

```json
{
  "organization_insights": {
    "possible_target_departments": [],
    "recommended_target_roles": [
      {
        "role": "",
        "reason": "",
        "priority": 1
      }
    ],
    "possible_decision_chain": [],
    "key_people_public_info": []
  }
}
```

### 说明
MVP 不一定能拿到真实关键人姓名，但至少要做到：
- 推荐找哪个部门
- 推荐找哪个角色
- 给出理由

---

## 6. 风险字段

```json
{
  "risk_signals": [
    {
      "risk_type": "",
      "risk": "",
      "description": "",
      "impact": "",
      "level": "medium",
      "source": "",
      "date": ""
    }
  ]
}
```

### `risk_type` 示例
- legal
- compliance
- financial
- reputation
- procurement_complexity
- organization_instability
- low_budget_probability
- unclear_demand

### MVP 注意
风险不一定都是“负面经营风险”，也包括**商务推进风险**。

例如：
- 采购流程复杂
- 决策链长
- 预算不明确
- 需求不稳定

---

## 7. 商务判断结果字段

```json
{
  "sales_assessment": {
    "customer_fit_level": "",
```

### 建议枚举
- `customer_fit_level`: high / medium / low
- `opportunity_level`: high / medium / low
- `follow_up_priority`: P1 / P2 / P3 / discard

### 作用
这是面向业务最关键的结果层。

### 说明

## 9. 证据与可追溯字段

因为后面商务问“你这个结论从哪来的”，要能追溯。

# 五、MVP 的判断逻辑建议

MVP 不要做太复杂的机器学习评分，先做**规则 + LLM 归纳**最稳。

---

## 1. 客户匹配度判断逻辑

### 输入因素
- 行业是否匹配
- 企业规模是否匹配
- 企业类型是否匹配
- 地域是否匹配
- 业务场景是否匹配

### 示例规则
- 行业匹配 + 有明确需求信号 + 公司规模合适 → 高匹配
- 行业匹配但无明显动态 → 中匹配
- 行业不匹配且无需求信号 → 低匹配

---

## 2. 商机强度判断逻辑

### 可用信号
- 扩张
- 招聘
- 融资
- 新业务
- 数字化转型表述
- 招投标
- 高管发言
- 已使用同类工具/竞品

### 示例
- 最近 3 个月连续有扩张、招聘、数字化信号 → high
- 只有基础信息无新动态 → low/medium

---

## 3. 优先联系人判断逻辑

根据**你方产品类型**映射角色。

### 例如
#### 如果你卖 CRM / 销售管理系统
- 首选：销售负责人 / 销售运营负责人
- 次选：信息化负责人 / COO

#### 如果你卖 AI 办公工具
- 首选：IT / 数字化负责人
- 次选：业务部门负责人 / 行政运营

#### 如果你卖培训服务
- 首选：HRD / 培训负责人
- 次选：业务部门负责人

---

## 4. 沟通切入点生成逻辑

切入点最好基于以下模板：

> 企业近期动作 + 潜在业务挑战 + 可带来的价值

例如：

- 企业近期扩张 → 团队协同复杂 → 切“流程效率”
- 企业招聘销售运营 → 销售体系建设中 → 切“客户管理标准化”
- 企业推新产品 → 获客和转化压力上升 → 切“线索管理/增长转化”

---

# 九、给你一个可直接开发的 Prompt 分层思路

为了避免一个大 Prompt 包打天下，我建议拆成 3 个任务。

---

## Prompt 1：企业信息抽取

目标：从原始文本抽取结构化字段。

输入：
- 官网介绍
- 新闻内容
- 招聘内容
- 工商信息

# 十、MVP 成功标准怎么定义

你做 MVP 时，一定要先定义“怎样算成功”。

建议用下面几个标准：

---

## 1. 效率指标
- 商务准备一个企业的时间，从 30 分钟降到 5 分钟以内

## 2. 可用性指标
- 输出内容中，商务认为“有用”的比例超过 60%

## 3. 结论可信度
- 每条关键结论都有依据和来源

## 4. 可执行性
- 商务愿意直接复制使用部分话术

## 5. 覆盖完整度
- 输出至少覆盖：
  - 企业是谁
  - 最近动态
  - 潜在需求
  - 找谁
  - 怎么说

# 十三、给你一个最终建议：MVP 先用“行业 + 产品类型”做限定

为了让结果更准，建议 MVP 一开始不要“面向所有 ToB 场景通吃”。

你可以限定成下面任一方向：

### 方向 A：面向卖 SaaS / CRM / 销售管理工具的商务
优点：
- 需求信号比较容易定义
- 联系人相对清晰
- 话术模板更容易标准化

### 方向 B：面向卖数字化/AI/效率工具的商务
优点：
- 当前市场热
- 需求场景广
- 招聘/转型/提效信号比较容易关联

### 方向 C：面向人力/培训类商务
优点：
- 背调重点比较稳定
- 角色相对明确

如果你完全没行业经验，**我最建议先从 A 或 B 切**。
